# 접근통제(접근제어Access Control)
## 접근통제의 정의 : 접근통제는 적절한 권한을 가진 인가자만이 특정 시스템이나 정보에 접근할 수 있도록 통제하는 것
1. 접근 : 주체와 객체 사이의 정보의 흐름
2. 통제 : 조직의 목표가 달성될 것이며, 바람직하지 못한 사건들이 예방, 적발 및 교정될 것이라는 합리적인 수준의 보증을 제공하기 위해 설계된 정책, 절차, 실무관행, 그리고 조직구조

- 접근활동 : 활동의 종류(CRUD) Create, Read, Update, Delete 
- 접근통제의 과정 : 접근 요청시 -> 접근거부, 접근허용
- 접근통제의 3단계 : 식별(identification) -> 인증(authentication) -> 인가(authorization)

## 접근통제의 개념 , 접근통제 = 인증 + 인가 
1. 식별 : 자신이 누구인지 밝히는 것, 사례: 사용자명, 계정번호
2. 인증 : 자신의 신원이 올바른지 증명하는 활동 또는 과정 , 사례 : 패스워드, 스마트카드, 생체인증
3. 인가 : 사용자의 특정 객체에 대한 접근을 허용하는 과정 , 사례 : 접근통제목록(ACL Access Control List), 보안등급

## 개인식별과 사용자 인증
1. 개인식별(신분확인) : 주체가 공공정보를 입력하여 기존에 내장된 데이터베이스의 정보와 일치하는지를 확인하는 단계
- 공공정보 : 사용자이름 , 개인식별번호(PIN) , 스마트카드 , 계좌번호 등
2. 사용자 인증 : 신분의 검증을 의미하며, 사용자가 본인임을 증명하는 단계(예 : 패스워드, 일회용 패스워드, 암호키, 전자서명 등)

# 중간고사 문제: 식별과 인증의 비교
- 식별 : 1:N , 등록된 입력정보 , 중요한요소는 유사도 , 구현과정이 복잡하다.
- 인증 : 1:1 , 해당된 입력정보 , 중요한 요소는 일치도 , 구현과정이 간단하다.

## 사용자 인증의 유형
1. 지식기반인증 : 사용자가 알고 있는 것을 보여줌
2. 소유기반인증 : 사용자가 소유하고 있는 것을 보여줌
3. 존재기반인증 : 사용가는 그를 나타내는 것을 보여줌
4. 행위기반인증 : 사용자는 그가 하는 것을 보여줌
- 강한 인증을 위한 혼합인증 : 2-Factor (위에 4종류 중에 2가지 요소를 합하는 것)

# 접근 통제(Access Control)방법의 고찰(고찰 : 어떤것을 깊게 생각하고 연구함.)
## 효과적인 접근을 통제하기 위한 방법이 많이 연구 되어왔다.
## 접근 통제의 유형
1. 임의적 접근통제 - 객체적 : 시스템 자원에 접근하는 사용자의 접근모드 및 모든 접근제한 조건 등을 정의하는 접근 통제 정책 (신원기반 접근통제, 사용자기반 접근통제:시스템접근시 사용자의 접근제한), 혼합방식 접근통제
2. 강제적 접근통제 - 주체적 : 시도된 접근요청에 정의된 규칙에 대응시켜 검사함으로써 불법적 접근을 방어하는 접근통제
(규칙기반 접근통제 : 규칙대응 하는 접근에 따른 방식, 관리기반 접근통제)
3. 비임의적 접근통제 -주체와 객체로 역할 분담 : 시스템 보안요구를 나타내는 요구명세서로부터 출발하여 간결한 기능적 모델을 표현하는 접근통제(역할기반 접근통제, 직무기반 접근통제, 래티스 접근통제-현재 제일 많이 쓰고 있다, 기능이 다 다름(효율적))

## 접근 통제 해석
1. 강제적 접근통제(MAC:Mandatory Access Control) : 주체의 권한에 근거하여 접근제한
- 객체에 접근할 때 사전에 규정된 접근제한과 비교하여 그 제한에 만족한 주체에게만 접근권한 부여함.
- 기밀성 위주로 보안 레벨을 부여하여 제한함(비밀등급, 허가등급 등)
2. 임의적 접근통제(DAC:Discretionary Access Control) : 개인, 그룹의 identity에 근거하여 접근 통제
- 그룹의 신원에 의한 관리자의 개입 없이 자율적 판단에 따라 접근권한부여(기업적인 내부 환경이나 상업적인 환경에서 활용)
3. 역할기반 접근통제(RBAC : Role Based Access Control) : 조직 내 역할 기반 자원 접근 허용(DAC + MAC)
- 1과 2의 대안으로 각광받고 있음.
- 주체와 객체의 상호작용에 의한 유연성 접근통제
- 조직 내 사용자가 가진 역할에 따라 객체에 대한 접근권한 지정 및 허용(병원이나 은행에서 특정한 역할 부여)

## 규칙기반 접근 통제(RBAC:Rule Based Access Control)
- 접근 수준과 허용 등급으로 특정한 규칙을 기반으로 접근통제
- 접근 수준과 등급은 관리자에 의해 설정 및 변경이 가능
- 대규모 시스템의 환경에서는 운영상의 과부하 부담이 발생

## 래티스 기반 접근통제(LBAC : Lattice Based Access Control)
- 어떠한 주체가 어떤 객체를 접근하거나 할 수 없는 경계를 지정하는 방식(주체에 대한 상위 경계부터 하위 경계까지 권한을 제공함)

# 접근통제관리(Access Control Administration)
1. 중앙집중 접근통제관리(예:사용감사기록)
 - RADIUS : UDP, 인증과 인가가 통합
 - TACACS : TCP, TACACS 헤더만 빼고 전체 패킷을 암호화 인증과 인가가 분리 됨.
2. 분산 접근통제관리
 - SSO : Kerberos, SESAME
 - Security Domain : 도메인 내의 고유한 보안정책이 적용된 환경

# 접근통제보안모델
## 상태머신 모델 : 가장 기초적, 시스템의 불완전 상태방지 관념적 모델(관념 : 사람의 마음 속에 나타나는 표상·상념·개념 또는 의식내용을 가리키는 말)
## 벨 라파둘라 모델(BLP Bell Lapadula Model)
  - 정보흐름에 대한 최초의 수학적 모델
  - 시스템 내부의 정보에 대한 기밀성, 등급에 따른 해석 -> 강제적 접근통제
  - 군사용 보안구조의 요구사항을 만족
  - 기밀성 유지에만 초점을 ㄷㅁ
  - 극비-비밀-미분류로 나누고 있음
  - 벨 라파둘라 모델의 No read-up , No Write-down
  - No read-up : 보안수준이 낮은 주체는 보안수준이 높은 객체를 읽을 수 없는 정책
  - No write-down : 보안수준이 높은 주체는 보안수준이 낮은 객체에 기록해서는 안되는 정책(비밀등급 이하의 정보를 수정하지 못하게 하는 보안 정책)
## Biba 모델(Biba Integrity 모델)
- 벨 라파둘라 모델에서 불법수정방지 내용을 추가로 정의한 무결성을 보안한 모델
- No read-down , No write-up 
- No read-down : 보안수준이 높은 주체는 보안수준이 낮은 객체를 읽을 수 없는 정책
- No write-up : 보안수전이 낮은 주체는 보안수준이 높은 객체에 기록해서는 안되는 정책

### 벨 라파둘라 모델과 비바 모델의 비교=> 둘다 책임추적성을 해결할 수 없다.
1. 벨 라파둘라 모델 : 정보흐름에 대한 최초의 수학적 모델 / 기밀성 / 극비,비밀,미분류로 보안기준이 나뉨/ no readup,no writedown / 군사용도
2. Biba 모델 : 벨 라파둘라 모델에서 무결성을 보완한 통계적 모델 / 무결성+기밀성 / 통계적근거 보안기준 / no writeup, no readdown / 공지사항,일반용도

## Clark and Wilson 모델
- Biba모델의 개선, 무결성 기능
- 상업 환경에 적합하게 개발된 불법수정방지를 위한 보안모델
- 금융자산의 관리나 회계 등의 내용을 담는 환경에서 기밀성보다 무결성 강조

## Lattice 모델 (가장 많이 사용됨.)
- 모든 자원과 그 자원을 이용하는 모든 사용자는 정보의 어느 한 비밀 구분에 속해야 함을 전제로 함.
- 주체 및 객체에 보안 클래스를 부여하여 정보흐름을 통제하고 보안 클래스를 중요도에 따라 구분하여 선형으로 배열시킨 구조임.

## Access Matrix 모델
- ACL(Access Control List)에 대한 주체와 객체의 접근허용권한을 주체로 행으로 객체는 열로 테이블로 나타낸 것
- 시스템의 사용자가 많을수록 접근 매트릭스는 일일이 목록을 만들어야 함으로 복잡함.
- 책임추적성이 어려움

## Take-Grant 모델
- 이 모델은 인가를 누가 어떻게 언제 해주어야 하는가에 대한 것을 규정한 모델임
- 인가승인과 취소의 결과 연구

## 기타 모델
- 고수준 사용자와 저수준 사용자로 상호 간섭배제 비간섭 모델, 정보흐름모델, 만리장성모델

# 접근토엦 원칙
1. 직무분리(Separation of Duty)
2. 최소권한(Least Privilege Policy)

## *인가 : 권한부여
- 식별과 인증의 단계를 거친 후 인가라는 것이 있다. 이는 접근을 요청받은 시스템은 주체가 요구하는 작업을 하는 데 다시 필요한 권한을 가지고 있는지 관리부서에서 판단해야 한다.
- 사용자가 식별과 인증단계로 검증되어 시스템에 원하는 작업에 대한 권한부여를 허가 받았다 하더라도, 파일에 대한 열람허용여부는 접근기능에 근거하여 특정시간 , 업무유형 등에 접근이 이루어진다.
- 권한부여 기술 : 접근기준 목록:역할,그룹,권한부여 등 / 기본적인 접근금지 / 최소권한 원칙

## *접근통제 3단계
1. 식별(1:N) 본인이 누구라는 것을 시스템에 밝히는 것 / 사용자명,계정번호 등
2. 인증(1:1) 주체의 신원을 검증하기 위한 사용 증명 활동 / 패스워드,토큰 등
3. 인가 인증된 주체에게 접근을 허용하고 특정 업무를 수행할 권리를 부여하는 과정 / 접근제어목록 , 보안등급 등

## 생체인식이란? 살아있는 사람의 신원을 생리학적 특징 또는 행동학적 특징을 기반으로 인증하거나 인식하는 자동화된 방법
## 생체인식 기술의 종류
1. 신체적(생리학적) 특징 이용: 지문인식, 얼굴, 홍채,망막,손모양 등 인식
2. 헹동학적 특징 이용 : 서명,필체,음성,걸음걸이, 키보드타이핑 등 

## 생체인식 시스템의 요구되는 특징
1. 이상적인 신체 특징 
 - 보편성 : 누구나 갖고 있는 특성을 이용해야함.
 - 유일성 : 각 개인을 구별할 수 있는 고유한 특성이 있어야 함.
 - 영속성 : 변하지 않고 변경이 불가능해야 함
 - 정량성 : 센서에 의한 획득과 정량화가 쉬어야함.
2. 설계 시 고려사항 
 - 성능 : 정확도, 속도, 리소스 요구 정도
 - 수용 : 사람의 수용정도
 - 오용 : 시스템을 속이기 쉬운 정도 

## 효율성과 수용성
### 생체인식의 주요 성능 측정방법
1. 거부오류비율(FRR : False Rejection Rate)
2. 허용오류비율(FAR : False Acceptance Rate)
3. 교차에러비율(CER : Cross Error Rate)
